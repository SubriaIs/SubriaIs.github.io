Customer Support Management

Overview


Objective: This project is a customer support management system designed to streamline interactions between users and support staff. It features a role-based access structure with three main user types: Customer, Staff, and Admin. Each role has specific permissions—Customers can create and view support reports, Staff can handle and update reports, and Admins can manage users and oversee the entire system. The application includes a secure login and signup mechanism to ensure proper authentication and access control. The goal is to provide an efficient and organized way to handle customer issues, assign tasks to support staff, and maintain a record of all support activities.


Duration: June 2024 - August 2024


Team Size: Solo project

Tech Stack


• Backend: Java


• Frontend: JavaFx


• Database: Mysql

Key Features


• Authentication System :
-Provides secure login functionality for all user roles (Customer, Staff, Admin) using a unique user ID and password.
-Signup feature for new Customers.


• Customer Features :
-Create and submit new support reports.
-View and search previously submitted reports in a table view.


• Staff Features :
-View reports assigned to them.
-Update report status and resolution details.
-Search assigned reports using various criteria


• Admin Features :
• Person Management:
-Create new users (Customer, Staff, Admin)
-View and search user details

• Customer Support (Cases) Management:
-View all reports (created and resolved)
-Search reports based on different parameters

• Role-Based Access Control :
-Each role has specific permissions tailored to their responsibilities


• Search Functionality :
-Advanced filtering and search options for reports and users


• Dependency Requirement :
-Requires the customer_support_mgmt_lib project (available on GitHub) to function properly; must be downloaded and integrated before use.
